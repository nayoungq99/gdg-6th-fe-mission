컴포넌트: 화면(UI) 구성하는 작은 단위.
-> 독립적으로 동작하며 여러 컴포넌트 만들어 하나의 화면 만듦.

- 컴포넌트 사용 이점

1. 재사용성: 같은 UI를 여러 곳에서 반복해 사용 가능.
2. 유지보수에 좋음: 코드가 나누어져 있어 수정할 때 필요한 부분만 수정해 사용하면 됨.
3. 가독성: 하나의 파일에 모든 코드를 작성하는 것보다 구조가 명확해서 가독성이 향상됨.
4. 협업: 여러 사람이 나누어 코드를 작성할 때 용이함.

- 컴포넌트를 나누는 기준

1. 기능 단위: 예시 -> Narbar, Content, Footer
2. UI 구조: 화면에서 구분되는 영역별로 나눔.
3. 재사용 여부: 여러 곳에서 사용할 수 있는 요소는 별도의 컴포넌트로 나누는 것이 좋음.
4. 역할 명확성: 하나의 컴포넌트는 하나의 역할만 가지도록 함.
