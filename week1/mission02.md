- HTML 기본 문법
  :요소의 중첩 사용이 가능하다. 이때 부자관계가 성립된다.
  -> 예시: <html>
  <body>
  <h1>hello</h1>
  <p>hi</p>
  </body>
  -> html 요소는 body 요소를 포함하고 body 요소는 h1,p 요소를 포함한다. 알아보기 쉽게 들여쓰기 사용.
  :content를 가질 수 없는 요소를 빈 요소, 어트리뷰트만 가질 수 있음.(br, hr, img, input, link, meta)
  :어트리뷰트는 요소의 성질, 특성 정의하는 명세. 요소의 추가적 정보(이미지 파일 경로, 크기 등)제공.
  -> 예시: <img src="html.png">, src: 어트리뷰트명, "html.png": 어트리뷰트 값
  :글러벌 어트리뷰트, 모든 html 요소가 공통으로 사용 가능.(id, class, hidden, lang, style, tabindex, title)

- 시맨틱 요소와 검색 엔진
  :시맨틱 웹, SEO(검색엔진 최적화)같은 마케팅 도구 사용해 본인이 원하는 구조로 웹사이트 조정하기도 한다.
  :크롤링 -> 검색엔진이 로봇 프로그램 이용해 매일 전세계의 웹사이트 정보를 수집하는 것.
  :인덱싱 -> 검색 사이트 이용자가 검색할 키워드 예상해 대응하는 인덱스를 만들어두는 것.
  :인덱스 생성할 때 사용되는 정보 = 검색 로봇이 수집한 정보 = 웹사이트 html 코드
  :<h1>hello</h1> -> header 중 가장 상위 레벨 의미, 개발자가 의도한 요소의 의미가 정확함. 코드의 가독성 좋고 유지보수 쉬움.
  :시맨틱 태그 -> 브라우저, 검색엔진, 개발자 모두에게 콘텐츠의 의미 명확히 설명하는 역할.
  (header, nav, aside, section, article, footer)
  :시맨틱 웹 -> 웹에 존재하는 수많은 웹페이지들에 메타데이터 부여.
  웹페이지를 의미와 관련성 가지는 거대한 데이터베이스로 구축하고자하는 발상.
  :시맨틱 요소(form, table, img, content의 의미 명확히 설명),
  넌시맨틱 요소(div, span, content에 대해 설명X)

- 웹페이지 구성하는 기본 태그
  : 문서 형식 정의 -> 출력할 웹 페이지 형식을 브라우저에게 전달.
  문서 최상위 위치, 대소문자 구별X
  예시 -> <!DOCTYPE html>
  :html tag
  :head tag -> 메타데이터 포함 위한 요소, 웹페이지에 단 하나만 존재.
  (title tag, style tag, link tag(html과 외부 css파일 연계), script tag, meta tag(description(웹페이지 설명), keywords(seo 위한 검색엔진이 사용할 키워드 정의), author(웹페이지 저자), 기타 메타데이터 정의에 사용))
  :body tag

-텍스트 관련 태그
:제목 태그 -> h1부터 h6까지 존재. 순서대로 중요도, 크기가 큼.
시맨틱 웹 의미 살려 제목 이외에는 사용x
:글자 형태 태그 -> b(bold), strong(bold, 의미론적 중요성 의미O), italic체, em(강조 필요할 때, 의미론적 중요성 의미O), small, mark(형광펜), del(글자 중간에 가로선), ins(밑줄), sub/ sup(글자 아래, 위)
:본문 태그 -> p(단락), br(강제 개행, 종료태그 없음! space와 enter로는 항상 1개의 공백, 연속적 공백 삽입하려면 "&nbsp;" 사용), pre(태그 내의 content는 작성된 그대로 표시), hr(수평줄), q(짧은 인용문, 큰따옴표로 감쌈), blockquote(긴 인용문, 들여쓰기함)

- HTML의 핵심 개념, Hyperlink
  :HyperText -> Hyper는 컴퓨터 용어, 정보가 다중 연결되어 있는 상태 의미.
  사용자가 원하는 순서대로 정보 취득할 수 있는 기능 제공.
  Hyperlink -> 한 텍스트에서 다른 텍스트로 건너뛰어 읽을 수 있게 하는 기능, a tag가 역할 담당
  :href 어트리뷰트 -> 이동하고자 하는 파일의 위치를 값으로.
  디렉터리 -> 파일과 다른 디렉터리를 갖는 파일 시스템 내의 존재물. 폴더.
  (루트 디렉터리, 홈 디렉터리, 작업 디렉터리, 부모 디렉터리)
  파일 경로 -> 파일 시스템에서 파일의 위치 나타내는 방법(절대경로(루트 디렉터리 기준으로 파일 위치 나타냄), 상대경로(현재 작업 디렉터리 기준으로 파일의 상대적인 위치))
  -> href 어트리뷰트에 사용 가능한 값 -> 절대 URL, 상대 URL, fragment identifier(페이지 내의 특정 id 갖는 요소에 링크, href="#top"), 메일, script
  :target 어트리뷰트: 링크 클릭했을 때 윈도우 어떻게 오픈할지 지정.
  -> \_self(현재 윈도우에서 오픈, 기본값), \_blank(새로운 윈도우, 탭에서 오픈)
  -> target ="\_blank" 사용해 외부페이지 오픈하는 경우, 이동한 외부 페이지에서 악의적인 페이지로 연결할 수 있다는 단점(피싱 공격)있음. 따라서 rel="noopener noreferrer" 추가해 대비함.

- JavaScript 기본 문법
  :블록 유효범위를 생성하지 않는다.

- 데이터 타입과 변수
  :원시 타입(변경 불가능한 값, 값에 의한 전달) -> 문자열
- undefined -> 선언 되었지만 값 할당하지 않은 변수 접근, 존재하지 않는 객체에 접근할 경우 반환되는 값. (자바스크립트 엔진에 의해 초기화)
- null 타입 확인할 때 일치연산자 사용.
- 동적 타입 언어라 자동으로 타입이 결정됨. 같은 변수에 여러 타입의 값을 할당할 수 있고 이를 동적 타이핑이라고 부름.
- 모든 선언문 호이스팅 됨. (모든 선언문이 해당 scope 선두로 옮겨진 것처럼 동작)
  선언 단계, 초기화 단계, 할당 단계, 함수 레벨 스코프(함수 내에서만 유효)

- 표현식과 연산자
  : 쉼표 연산자(왼쪽 피연산자부터 차례로 평가, 마지막 피연산자 반환)
  할당 연산자(좌항에 있는 변수에 할당. 부수 효과가 있다) -> =, +=, -=, \*=, /=, %=

- 제어문
  : 블록문을 하나의 단위로 취급함. 세미콜론 붙이지 않는다.
  continue 문은 반복문의 코드 블록 실행을 현 지점에서 중단. if문 내에서 실행해야 할 코드가 길다면 continue문을 사용하는 것이 가독성 면에서 좋음.
